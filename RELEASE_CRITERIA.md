# Release Criteria

This private workbench should not be made public directly.

A future public release should be generated from reviewed, sanitized artifacts only.

## v0.1 private target

```text
25–40 contributors
100–250 sources
500–1,500 atomic claims
25–40 claim families
100–300 evidence artifacts
10–20 fully audited high-risk claims
```

## Minimum viable private ledger

```text
10 contributors
5 sources per contributor
10 claims per source
≈ 500 claim-source observations
```

## v1 public release gates

Do not release publicly until:

```text
1. Public-bound repo has clean Git history.
2. No raw transcripts or long copyrighted excerpts are included.
3. No private notes are included.
4. No unreviewed unsafe medical claims appear as recommendations.
5. Methodology is documented.
6. Safety policy is documented.
7. Evidence taxonomy is documented.
8. Risk taxonomy is documented.
9. Source-quality taxonomy is documented.
10. At least 1,000 staged or reviewed claims exist.
11. At least 10 high-risk audit packets are complete.
12. Red-team sources exist for each high-risk family.
13. Public-safe wording exists for all protocol-admissible claims.
14. Protocol statuses are assigned.
15. Known limitations are stated.
16. License is selected.
17. Release notes are written.
18. Data validation passes.
```
